RCcar – informace o projektu

{# hlavní nadpis stránky #}

RCcar je školní projekt zaměřený na propojení webové aplikace, databáze a mikrokontroleru Arduino. Uživatel se přihlásí do webového rozhraní, vybere auto, může spustit a ukončit jízdu a zároveň ovládat zatáčení servomotoru a motor autíčka přes dashboard.

{# úvodní popis projektu stručně říká, o čem projekt je a co umí #}

Komunikace s Arduinem probíhá přes USB kabel pomocí sériové komunikace. Webová aplikace tedy neposílá příkazy přímo do auta, ale přes Python backend, který je následně odešle do Arduina.

Použité technologie

{# sekce s použitými technologiemi #}

Backend aplikace je napsaný v jazyce Python s použitím frameworku Flask. Databáze je řešena pomocí SQLite. Pro bezpečné ukládání hesel se používá Werkzeug, který hesla zahashuje. Komunikace s Arduinem probíhá přes knihovnu pyserial. Frontend je tvořen pomocí HTML, CSS a JavaScriptu. Arduino kód je napsaný v jazyce C++ pomocí Arduino knihoven.

Databáze projektu

{# sekce věnovaná databázi #}

Databáze projektu slouží k ukládání uživatelů, aut a jízd. Každá jízda je přiřazena jednomu uživateli a jednomu autu. Díky tomu lze evidovat, kdo s jakým autem jel a jak dlouho jízda trvala.

{# základní vysvětlení, co se v databázi ukládá #}

V databázi jsou použité tyto vztahy:

{# úvod k vypsání vztahů mezi tabulkami #}
  • 1:N mezi users a rides – jeden uživatel může mít více jízd
  • 1:N mezi cars a rides – jedno auto může být použité ve více jízdách
  • M:N mezi users a cars – tento vztah je realizovaný přes tabulku rides

Princip fungování aplikace

{# sekce, která popisuje tok práce v aplikaci #}

Uživatel se přihlásí do webové aplikace, vybere auto a v dashboardu může ovládat zatáčení a motor autíčka. Při stisku tlačítka Start se do databáze založí nová jízda. Při stisku Stop se jízda ukončí a dopočítá se její délka. Pokud se stránka zavře bez ukončení jízdy, zůstane v databázi záznam s nevyplněnou délkou jízdy, což značí nedokončenou jízdu.

Použité algoritmy a logika

{# sekce s logikou aplikace krok po kroku #}
{# větší řádkování kvůli lepší čitelnosti #}

1. Přihlášení uživatele
Zadané heslo se porovná s hashovaným heslem uloženým v databázi. Pokud se údaje shodují, vytvoří se uživatelská session.

{# login funguje přes porovnání hesla s hashem v databázi #}

2. Výběr auta
Uživatel si v dashboardu vybere auto, jehož id se uloží do session. Díky tomu backend ví, se kterým autem aktuálně pracuje.

{# výběr auta se uloží do session, aby ho backend znal i dál #}

3. Start jízdy
Po kliknutí na tlačítko Start se do tabulky rides vloží nový záznam obsahující uživatele, vybrané auto a čas začátku jízdy.

{# při startu jízdy vznikne nový záznam v tabulce rides #}

4. Stop jízdy
Po kliknutí na Stop se dopočítá délka jízdy jako rozdíl času konce a času začátku. Výsledek se uloží do databáze do sloupce duration_sec.

{# při ukončení jízdy se spočítá její délka a uloží do databáze #}

5. Ovládání auta
JavaScript pošle příkaz na Flask endpoint, Flask ho přes pyserial odešle do Arduina a Arduino podle příkazu nastaví servo nebo motor.

{# cesta příkazu: web -> JavaScript -> Flask -> pyserial -> Arduino #}
