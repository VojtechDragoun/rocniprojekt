RCcar – informace o projektu

RCcar je školní projekt zaměřený na propojení webové aplikace, databáze a mikrokontroleru Arduino. Uživatel se přihlásí do webového rozhraní, vybere auto, může spustit a ukončit jízdu a zároveň ovládat zatáčení servomotoru přes dashboard.

Hlavní funkce projektu

Projekt aktuálně umožňuje registraci a přihlášení uživatele, rozlišení běžného uživatele a admina, výběr auta, záznam jízd do databáze a ovládání serva přes Arduino. Admin má navíc možnost prohlížet a mazat vybraná data z databáze přes administrační rozhraní.

Co už projekt umí:

  • registrace a přihlášení uživatele
  • ukládání hashovaných hesel do databáze
  • role uživatel / admin
  • výběr auta v dashboardu
  • start a ukončení jízdy
  • ukládání jízd do databáze
  • ovládání servomotoru přes web
  • správa databázových dat v admin sekci

Použité technologie

Backend aplikace je napsaný v jazyce Python s použitím frameworku Flask. Databáze je řešena pomocí SQLite. Pro bezpečné ukládání hesel se používá Werkzeug, který hesla zahashuje. Komunikace s Arduinem probíhá přes knihovnu pyserial. Frontend je tvořen pomocí HTML, CSS a JavaScriptu. Arduino kód je napsaný v c++.

Použité jazyky a knihovny:

  • Python
  • Flask
  • Werkzeug
  • sqlite3
  • pyserial
  • HTML
  • CSS
  • JavaScript
  • C++ / Arduino

Databáze projektu

Databáze projektu slouží k ukládání uživatelů, aut a jízd. Každá jízda je přiřazena jednomu uživateli a jednomu autu. Díky tomu lze evidovat, kdo s jakým autem jel a jak dlouho jízda trvala.

V projektu jsou použité vztahy 1:N i M:N:

  • 1:N mezi users a rides – jeden uživatel může mít více jízd
  • 1:1 mezi username a id – každý jeden uživatel má svoje unikátní id
  • M:N mezi users a cars – jeden uživatel může mít více aut a jedno auto může mít více uživatelů

Princip fungování aplikace

Uživatel se přihlásí do webové aplikace, vybere auto a v dashboardu může ovládat zatáčení serva. Při stisku tlačítka Start se do databáze založí nová jízda. Při stisku Stop se jízda ukončí a dopočítá se její délka. Pokud se stránka zavře bez ukončení jízdy, zůstane v databázi záznam s nevyplněnou délkou jízdy, což značí nedokončenou jízdu.

Použité algoritmy a logika

1. Přihlášení uživatele
Zadané heslo se porovná s hashovaným heslem uloženým v databázi. Pokud se údaje shodují, vytvoří se uživatelská session.

2. Výběr auta
Uživatel si v dashboardu vybere auto, jehož id se uloží do session. Díky tomu backend ví, se kterým autem aktuálně pracuje.

3. Start jízdy
Po kliknutí na tlačítko Start se do tabulky jízd vloží nový záznam obsahující uživatele, vybrané auto a čas začátku jízdy.

4. Stop jízdy
Po kliknutí na Stop se dopočítá délka jízdy jako rozdíl času konce a času začátku. Výsledek se uloží do databáze do sloupce duration_sec.

5. Ovládání serva
JavaScript pošle příkaz na Flask endpoint, Flask ho předá přes pyserial do Arduina a Arduino nastaví odpovídající úhel serva.
